## 提交需求

这是最主要的参与方式。你不需要会写代码，只需要有想法。

### 第一步：想清楚你要什么

问自己几个问题：

- 这个东西解决什么问题？
- 谁会用它？
- 现在有没有类似的？如果有，哪里不够好？

### 第二步：提交 Issue

**[👉 点击这里提交需求](https://github.com/openvibelab/openvibelab/issues/new?template=project-request.yml)**

填写以下内容：

- **项目名称** — 给你的想法起个名字
- **一句话描述** — 用一句话说清楚它是什么
- **详细说明** — 它应该有哪些功能？解决什么问题？
- **参考** — 有没有类似的产品可以参考？（可选）

### 第三步：等待反馈

提交后我会：

1. 在 Issue 下回复，确认收到
2. 评估可行性和工作量
3. 标记状态（排队中 / 需要讨论 / 暂不做）
4. 排期后开始开发

### 如何让你的需求更快被做？

- **描述清晰** — 越具体越好
- **获得投票** — 让更多人给你的 Issue 点 👍
- **积极互动** — 在 Issue 下补充细节、回答问题

## 为项目贡献代码

如果你想给某个已有项目贡献代码：

1. Fork 对应的项目仓库
2. 创建你的分支：`git checkout -b feature/你的功能`
3. 提交修改：`git commit -m "feat: 添加了xxx功能"`
4. 推送分支：`git push origin feature/你的功能`
5. 创建 Pull Request

### Commit 消息规范

```
feat: 新功能
fix: 修复 bug
docs: 文档更新
style: 代码格式（不影响功能）
refactor: 重构
test: 测试相关
chore: 构建/工具相关
```

## 反馈与建议

用了某个项目之后，欢迎在对应的仓库提 Issue 反馈：

- 🐛 **Bug 报告** — 遇到了什么问题
- 💡 **功能建议** — 觉得还可以加什么
- 📝 **体验反馈** — 用起来感觉怎么样

---

## 行为准则

这是一个开放友好的社区，请：

- 尊重每一个参与者
- 友善地表达不同意见
- 不发布攻击性或歧视性内容
- 帮助新人融入社区
