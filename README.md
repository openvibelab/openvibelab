## 关于这个挑战

我正在进行一个 **100天 × 100个项目** 的 Web 开发挑战。

规则很简单：

- 每天（或每几天）完成一个完整的 Web 项目
- **所有需求来自社区** — 你觉得什么有用、有趣，就提什么
- **所有代码完全开源** — 放在 GitHub，随便看、随便学、随便改
- **所有项目部署在国内可访问的域名** — 不用翻墙，打开就能用
- 遇到好的项目，**考虑和社区一起正式发行上线**

> 这不只是我一个人的挑战，而是一个开放的社区实验。

## 💡 我想提需求！

有想法？太好了。提需求非常简单：

**[👉 点击这里提交你的项目需求](https://github.com/openvibelab/openvibelab/issues/new?template=project-request.yml)**

提需求时只需要说清楚：

1. **你想要什么** — 一句话描述
2. **它解决什么问题** — 为什么需要这个
3. **你期望的样子** — 越具体越好（也可以不具体，我来想）

> 不需要懂技术，不需要写代码，只要你觉得"要是有这个就好了"，就可以提。

## 🤝 参与方式

- **提需求** — [创建 Issue](https://github.com/openvibelab/openvibelab/issues/new?template=project-request.yml)，说说你想要什么
- **投票** — 给你喜欢的需求 Issue 点 👍，票多的优先做
- **贡献代码** — Fork 任意项目仓库，提 PR，欢迎一起写
- **反馈** — 用了项目之后说说体验，帮助改进
- **传播** — 分享给更多人，让社区更大

### 🔥 社区共建 & 项目孵化

这不只是我一个人写代码 — **每个项目都欢迎社区贡献 PR**。

如果你觉得某个项目有意思，想加个功能、改个 bug、优化下体验，直接提 PR 就行。好的贡献会被合并，优秀的贡献者会出现在项目的 Contributors 列表里。

**对于社区反响好、有潜力的项目，我们会认真打磨，考虑正式上线运营。** 从一个挑战项目变成一个真正的产品 — 如果你参与了贡献，我们一起做。
