## 这是什么？

我决定花 100 天，做 100 个项目。全部开源，全部可用。

但我不想自己闷头想做什么 — **我想做你们真正需要的东西**。

所以规则是这样的：

- **需求来自你** — 你生活中、工作中觉得"要是有这个就好了"的东西，告诉我
- **代码完全公开** — 每一行都在 GitHub 上，随便看、随便学、随便拿去用
- **做出来就能用** — 项目部署在 [openvibelab.com](https://openvibelab.com) 下，国内直接访问，打开就用
- **不限形态** — 以 Web 为主，但工具、脚本、小程序、API，什么好用做什么
- **好项目一起做大** — 社区反响好的项目，会认真打磨，上线小程序、App，甚至正式运营

> 这不是一个人的表演，而是一场开放的社区实验。<br>
> 你提想法，我写代码，我们一起把它变成真的。

---

## 为什么叫 Vibe Coding？

不是为了炫技，不是为了刷题，不是为了完成 KPI。

**Vibe Coding 是跟着感觉走** — 做有趣的东西，做有用的东西，做让人用了会笑的东西。快速原型，快速验证，快速上线。不纠结完美，先让它跑起来，再让它变好。

每个项目都是一次实验：能不能用最短的时间，做出一个真正有人用的东西？

## 🚀 项目进化路线

每个项目都会经历这样的旅程：

```
💡 社区提需求
  ↓
🔵 快速开发 MVP（1-3天）
  ↓
🟢 上线 Web 版，所有人可用
  ↓
👥 社区反馈 + PR 贡献
  ↓
⭐ 反响好？认真打磨 →  小程序 / App / 正式产品
```

**大部分项目会停在 MVP 阶段，这很正常。** 但那些真正击中需求的项目 — 用的人多、反馈好、有人愿意一起贡献 — 我们会把它从一个 demo 做成一个真正的产品。

参与过贡献的人，就是这个产品的共创者。

## 🤝 怎么参与？

**不会写代码？**
- 提需求 — [告诉我你想要什么](https://github.com/openvibelab/openvibelab/issues/new?template=project-request.yml)
- 投票 — 给你喜欢的需求点 👍，票多优先做
- 反馈 — 用完之后说说感受，好的坏的都要
- 传播 — 让更多人知道这件事

**会写代码？**
- 贡献 PR — 每个项目都欢迎，改个 bug 加个功能都行
- 一起做大 — 好项目会长期维护，贡献者就是核心成员

> 详细的贡献指南在这里：**[CONTRIBUTING.md →](./CONTRIBUTING.md)**
